## Основные директории

*   **`src/`** — корневой каталог исходного кода клиента.
   *   `src/game/` — состояние игры и обработка событий (authoritative host + события).
   *   `src/network/` — сетевой протокол, валидация и транспорт WebSocket.
   *   `src/questions/` — локальная база вопросов и детерминированный выбор по seed.
   *   `src/storage/` — локальная статистика (localStorage).
*   **`server/`** — минимальный relay‑сервер WebSocket (клиент‑серверный слой обязателен для браузера).

## Основные модули

1.  **GameState** (`src/game/state.ts`) — структура состояния (игроки, фаза, позиции, призовой фонд).
2.  **GameEvents** (`src/game/events.ts`) — события, применяемые к состоянию, и редьюсер.
3.  **GameLogic** (`src/game/logic.ts`) — базовые генераторы событий (старт/конец раунда, обновление приза).
4.  **Protocol** (`src/network/protocol.ts`) — формат сообщений клиент ↔ ведущий.
5.  **Validator** (`src/network/validator.ts`) — проверка входящих/исходящих сообщений.
6.  **WebSocketTransport** (`src/network/websocketClient.ts`) — WebSocket‑клиент с обработкой сообщений.
7.  **QuestionBank** (`src/questions/bank.ts`) — локальная база и генерация последовательности по seed.
8.  **LocalStats** (`src/storage/localStats.ts`) — локальная статистика и обновление результатов.

## Архитектурные принципы (согласно ТЗ)

*   Авторитетное состояние у ведущего: критические изменения рассылаются как события.
*   Все клиенты проверяют входящие сообщения и могут запросить snapshot при рассинхронизации.
*   Вопросы выбираются детерминированно по общему seed.
*   Для браузера необходим relay‑сервер (WebSocket — клиент‑серверный протокол).
